Hello, your Galaxy ToolFactory Docker container derived from quay.io/bgruening/galaxy:20.09 is running!

Important: first run steps After building or starting this container for the very first time, please wait until everything is fully installed - it can take 10-15 minutes or more. When the output from docker logs -f [container id]
settles down, please restart Galaxy using the supplied restart script:
docker exec [container id] sh /usr/local/bin/restartall.sh For some reason, the job runner will not work until after a restart.
When you login, there should be a new history containing the demonstration tools as jobs you can rerun and adjust
There is also a workflow available that will re-create that history and in the process, install those demonstration tools into the Galaxy instance you are using in the ToolFactory section (!)
Each demonstrates a variation of the patterns that can be generated and they can be re-run and adjusted to see how they look and work.
  • Hello runs a bash script with a single user supplied parameter - "World" for example. STDOUT goes to the history
  • Tacrev uses STDOUT and STDIN without parameters.
  • Two simple python scripts to reverse lines in a text file are driven with (pyrevargparse) argparse style and with (pyrevpos) positional parameters
  • The perl GC counter takes positional parameters and is taken straight from the Planemo documentation tool examples
  • The planemo advanced bwa example is shown in two versions. One uses the original Planemo tutorial code to demonstrate a command over-ride. An equivalent tool but using bash and positional parameters might be easier to maintain since it doesn't use cheetah markup.
  • The Sed example shows how unix utilities like sed - or appropriate Conda package - can be driven using positional parameters
